# Team Start Here

If you are joining the project, do these first:

1. Read `README.md` for the full picture.
2. Read `docs/ARCHITECTURE.md` for the system boundaries.
3. Read `docs/WORKSTREAMS.md` to understand the four team areas.
4. Read `docs/FINAL_PROTOTYPE_PLAN.md` so you know what the final product is supposed to become.
5. Read `CONTRIBUTING.md` before creating your first branch.
6. Run `.\run_tests.ps1` and `.\run_demo.ps1 -Sample 1` so you know the current baseline.

## Suggested Role Split For Four People

Use the four workstreams in `docs/WORKSTREAMS.md`:

- Person 1: System Architecture And Orchestration
- Person 2: Agents And Tools
- Person 3: Data / RAG / API
- Person 4: Training, Eval, Demo, Results

This is a better fit for the project story you want to tell and maps more closely to the final presentation.

## Before You Start Coding

- Pick a small task.
- Check whether it belongs to your workstream.
- Say which files you expect to touch.
- Create a branch.
- Keep your first PR small.

## When You Are Unsure

Prefer adding a short note in docs or opening a draft PR over making a large architectural guess alone.
